# Zone Analysis Reference

Quick reference for analysing accounts and determining appropriate actions based on AdPulse zones.

---

## Zone Determination Matrix

| Budget Pacing | Performance | Zone | Priority |
|--------------|-------------|------|----------|
| >110% (Over) | >110% CPA or <90% ROAS (Poor) | 🔴 Red | CRITICAL |
| 90-110% (On Pace) | >110% CPA or <90% ROAS (Poor) | 🟠 Orange | High |
| <90% (Under) | ≤110% CPA or ≥90% ROAS (Good) | 🟡 Yellow | Medium |
| >110% (Over) | ≤110% CPA or ≥90% ROAS (Good) | 🔵 Blue | Low |
| 90-110% (On Pace) | ≤110% CPA or ≥90% ROAS (Good) | 🟢 Green | Maintain |

---

## 🔴 Red Zone Analysis

### Characteristics
- **Budget**: Overspending (>110% of target pace)
- **Performance**: Poor (CPA >110% of target OR ROAS <90% of target)
- **Urgency**: CRITICAL - Same day action required

### Common Causes
1. Broad match keywords capturing irrelevant traffic
2. Competitor activity driving up CPCs
3. Landing page issues killing conversion rate
4. Seasonal demand shift not anticipated
5. Budget increased without performance monitoring
6. Smart bidding targeting wrong conversions
7. Geographic expansion performing poorly

### Priority Actions
1. **Immediate**: Reduce campaign budgets by 20-30%
2. **Same Day**: Run search terms report, add negative keywords
3. **24 Hours**: Pause highest-cost, zero-conversion keywords
4. **48 Hours**: Review landing pages for issues
5. **Week 1**: Test new ad copy with different messaging

### Expected Resolution Time
- Spend control: 1-2 days
- Performance improvement: 1-2 weeks

### Key Metrics to Track
- Daily spend vs target
- Search term quality
- Landing page conversion rate
- Impression share (budget lost)

### Execution Checklist (from AdPulse whitepaper)

Emit this checklist verbatim in the report when the account lands in Red. Stabilise the patient: check vitals, treat life-threatening issues first, then stabilise.

**One-off**
- [ ] Budget reduced to 80-90% of ideal daily calculation
- [ ] Budget allocation tweaked to favour best performers
- [ ] Conversion actions are firing
- [ ] Geo-targeting tight enough
- [ ] Search partners performance checked
- [ ] Ad extensions - all relevant types exist
- [ ] Search term audit - wasted spend focus
- [ ] Blocked search terms checked
- [ ] Competitor analysis - check ads and special deals
- [ ] Best campaigns - can they be expanded? (duplicate with different bid strategy / match type upgrade / demographics / audiences / geo)

**Daily**
- [ ] Budget reduced to 80-90% of ideal daily calculation
- [ ] Budget allocation tweaked to favour best performers
- [ ] All ads approved
- [ ] No 404 errors on landing pages

**Weekly**
- [ ] Budgets lasting all day - adjust bids down if not
- [ ] Search terms - wasted spend focus
- [ ] Check troubleshooter to find weak metrics

**Monthly**
- [ ] Check performance & add bid modifiers on anomalies (devices / locations / audiences / demographics)
- [ ] Pause/exclude poor performers (landing pages / ad groups / ads / keywords / search terms)
- [ ] Bid strategy - change/experiment?
- [ ] Landing page speed
- [ ] Bounce rate or time-on-page metrics

## 🟠 Orange Zone Analysis

### Characteristics
- **Budget**: On pace (90-110% of target)
- **Performance**: Poor (CPA >110% of target OR ROAS <90% of target)
- **Urgency**: High - Address within 2-3 days

### Common Causes
1. Ad copy not resonating with audience
2. Keywords too broad or poorly targeted
3. Landing page message mismatch
4. Competition has improved their offering
5. Wrong audience or demographic targeting
6. Ad fatigue from unchanged creative
7. Conversion tracking issues (false positive on pacing)

### Priority Actions
1. **Fix performance first** - Do not scale what isn't working
2. **Audit keywords** - Identify low-converting, high-spend keywords
3. **Review ad copy** - Test new headlines and descriptions
4. **Check landing pages** - Ensure message matches ads
5. **Review audience** - Check device, demographics, geography

### Expected Resolution Time
- Diagnosis: 2-3 days
- Full improvement: 1-2 weeks

### Key Metrics to Track
- Quality Score trends
- Click-through rate by ad
- Conversion rate by landing page
- Search term relevance

### Execution Checklist (from AdPulse whitepaper)

Emit this checklist verbatim in the report when the account lands in Orange. Flip the Red Zone priorities: performance fixes on top, budget increases only once performance is back on track.

**One-off**

- [ ] Feed issues checked
- [ ] Geo-targeting tight enough
- [ ] Search partners performance checked
- [ ] Ad extensions - all relevant types exist

- [ ] Blocked search terms checked
- [ ] Competitor analysis - check ads and special deals
- [ ] Best campaigns - can they be expanded? (duplicate with different bid strategy / match type upgrade / demographics / audiences / geo)

**Daily**
- [ ] Budget allocation tweaked to favour best performers
- [ ] All ads approved

**Weekly**
- [ ] Budgets lasting all day - adjust bids down if not
- [ ] Search terms - wasted spend focus

**Monthly**

## 🟡 Yellow Zone Analysis

### Characteristics
- **Budget**: Underspending (<90% of target pace)
- **Performance**: Good (CPA ≤110% of target OR ROAS ≥90% of target)
- **Urgency**: Medium - Scaling opportunity, address within 1 week

### Common Causes
1. Keyword targeting too narrow
2. Bids too low to win auctions
3. Budget capped on well-performing campaigns
4. New campaigns still ramping up
5. Ad schedule limiting impressions
6. Geographic targeting too restrictive
7. Audience size too small

### Priority Actions
1. **Scale carefully** - Increase budgets by 10-20% increments
2. **Expand keywords** - Add related terms, test broader matches
3. **Raise bids** - Improve impression share on winners
4. **Extend schedule** - Add more hours if appropriate
5. **Add campaigns** - Target adjacent services or locations

### Expected Resolution Time
- Initial scaling: 3-5 days
- Full budget utilisation: 2-3 weeks

### Key Metrics to Track
- Impression share (opportunity)
- Lost IS due to budget
- Lost IS due to rank
- Search volume available

### Execution Checklist (from AdPulse whitepaper)

Emit this checklist verbatim in the report when the account lands in Yellow. Easy zone: increase budgets where impactful, expand existing campaigns, then add new campaign types/networks.

**One-off**
- [ ] Budget allocation - increase where impactful
- [ ] Raise bids in best campaigns
- [ ] Blocked search terms checked
- [ ] Turn on search partners?
- [ ] Ad extensions - all relevant types exist
- [ ] Best campaigns - can they be expanded? (duplicate with different bid strategy / match type upgrade / keyword expansion / new ad formats / demographics / audiences / geo)
- [ ] Remarketing campaigns?
- [ ] Pay-for-conversions (display)?
- [ ] Add new campaign types?
- [ ] New networks (Bing)?

**Daily**
- [ ] Budget allocation - ensure top performers are not limited
- [ ] All ads approved
- [ ] No 404 errors on landing pages

**Weekly**
- [ ] Budgets lasting all day? Increase budgets if not

**Monthly**
- [ ] Check performance & add positive bid modifiers on anomalies (devices / locations / audiences / demographics)
- [ ] Bid strategy - change/experiment?

## 🔵 Blue Zone Analysis

### Characteristics
- **Budget**: Overspending (>110% of target pace)
- **Performance**: Good (CPA ≤110% of target OR ROAS ≥90% of target)
- **Urgency**: Low - Decision needed, not crisis

### Common Causes
1. Strong seasonal demand
2. Successful campaign expansion
3. Lower competition driving down CPCs
4. Budget set too conservatively
5. Recent performance improvements
6. New service offering gaining traction

### Priority Actions
1. **Communicate with client** - They may want to capitalise
2. **Request budget increase** - If client can invest more
3. **If no more budget** - Reduce lowest performers
4. **Pace the budget** - Spread remaining across month
5. **Protect winners** - Don't cut campaigns driving results

### Expected Resolution Time
- Decision: 1-2 days
- Implementation: Immediate once decided

### Key Metrics to Track
- Projected end-of-month spend
- Performance by campaign (to protect best)
- Client communication timing
- Available budget flexibility

### Execution Checklist (from AdPulse whitepaper)

The whitepaper gives Blue no checklist - the actions are sequential and run in order until spend is back in line. Emit these as ordered steps in the report:

1. [ ] Ask the client for more budget (performance is good - more leads/sales is an easy conversation)
2. [ ] Decrease bids, starting with poorer performers (10%/day until spend drops)
3. [ ] Decrease budget on poorer performers if bids alone do not pull spend back
4. [ ] Turn off worst-performing keywords / ad groups / campaigns as a last resort

## 🟢 Green Zone Analysis

### Characteristics
- **Budget**: On pace (90-110% of target)
- **Performance**: Good (CPA ≤110% of target OR ROAS ≥90% of target)
- **Urgency**: Low - Maintenance and testing mode

### Common Causes
1. Well-optimised account
2. Stable market conditions
3. Effective previous optimisations
4. Good ad-to-landing-page alignment
5. Appropriate targeting
6. Correct bid strategy

### Priority Actions
1. **Test new strategies** - Use stability to experiment
2. **Incremental optimisation** - Small improvements
3. **Monitor competitors** - Stay ahead of changes
4. **Plan for next month** - Anticipate seasonal shifts
5. **Document success** - Create templates for other accounts

### Expected Resolution Time
- Ongoing maintenance mode
- Weekly check-ins sufficient

### Key Metrics to Track
- Competitor impression share changes
- Quality Score stability
- Seasonal forecast alignment
- New keyword opportunities

### Execution Checklist (maintenance mode)

Green has no remedial checklist - the account is healthy. Emit these maintenance items in the report:

- [ ] Test a new strategy or experiment (use the stability to learn)
- [ ] Incremental optimisation - small, low-risk improvements
- [ ] Monitor competitor impression share and offers
- [ ] Plan for next month / anticipate seasonal shifts
- [ ] Document what is working as a template for other accounts

## Quick Decision Framework

```
START
  │
  ├─ Is budget >110% of pace?
  │   │
  │   ├─ YES → Is performance poor?
  │   │         ├─ YES → 🔴 RED (CRITICAL)
  │   │         └─ NO  → 🔵 BLUE (Low)
  │   │
  │   └─ NO → Is budget <90% of pace?
  │           │
  │           ├─ YES → Is performance good?
  │           │         ├─ YES → 🟡 YELLOW (Medium)
  │           │         └─ NO  → Check data accuracy
  │           │
  │           └─ NO → Budget is 90-110% (On Pace)
  │                   │
  │                   ├─ Is performance good?
  │                   │   ├─ YES → 🟢 GREEN (Maintain)
  │                   │   └─ NO  → 🟠 ORANGE (High)
```

## Using This Template

1. Gather performance data
2. Calculate budget pacing percentage
3. Calculate performance vs target
4. Use matrix to determine zone
5. Review relevant zone section for actions
6. Present prioritised recommendations
7. **Emit the matched zone's Execution Checklist in the report** (only the one zone that applies, not all five)
8. Wait for user approval before proceeding
