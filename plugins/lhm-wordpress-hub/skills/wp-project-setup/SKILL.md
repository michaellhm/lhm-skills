# Project Setup

Create the canonical folder structure for a new WordPress website project. The plugin convention separates shared client-level artefacts (cross-project reusable) from workflow-specific artefacts. This skill handles both layers.

## Step 1: Detect Project Location

Use the `AskUserQuestion` tool: "Where are we setting up this project?"

Options:
- **Inside an existing client folder** — current directory is already the client root (look for `client_profile.md`, `playbook.md`, or sibling `gmb/`, `landing-pages/` folders)
- **Create a new client folder** — ask for the client name, create `./[client-slug]/`
- **Project root is current directory but client root is one level up** — current dir is the wordpress folder

Detect intelligently:
- If CWD contains `client_profile.md` or `playbook.md` → current is client root, create `wordpress/` subfolder
- If CWD contains a `wordpress/` folder → already set up, ask if user wants to reset or work elsewhere
- If CWD has sibling `gmb/` or `landing-pages/` folders → current is client root
- Otherwise → ask the user explicitly via AskUserQuestion

## Step 2: Create Client-Root Scaffolding (if not present)

Check whether these files exist at the client root:

- `client_profile.md`
- `playbook.md`
- `website-brief.md`
- `clarifications.md`
- `design/` directory

If any are missing, create them with the templates in Step 4. Inform the user that Phase 1 (Steps 1.1–1.4) must populate these documents before Phase 2 can begin.

## Step 3: Create WordPress Subtree

Create `[client_root]/wordpress/` containing:

````
wordpress/
  seo/
    page_briefs/
  content/
    services/
    locations/
    supporting/
  prototype/
  wp/
    theme/
    blocks/
  qa/
    visual_qa_reports/
  docs/
    superpowers/
      specs/
      plans/
````

Do NOT create `website-project-management.md` here — that file is generated at the end of Phase 1 Step 1.5 by `wp-project-manager` after the Superpowers spec and plan exist.

## Step 6: Confirm and Hand Off

Tell the user:

> Project structure created. Here's what's next:
>
> 1. **Phase 1: Client Onboarding & Strategy** — Run the strategy session, generate the Campaign Playbook (Step 1.3) and Website Brief (Step 1.4), then run the Superpowers skill to generate the spec, plan, and PM doc (Step 1.5).
> 2. After Step 1.5 completes, the `wp-project-manager` skill will be invoked to create `wordpress/website-project-management.md`.
>
> Run `wp-start` when you're ready to begin.
