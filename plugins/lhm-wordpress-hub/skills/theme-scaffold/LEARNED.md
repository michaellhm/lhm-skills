- (2026-06-04) Dashboard-controllable menus in a block theme need a CLASSIC menu (`wp_nav_menu`) + a custom `Walker_Nav_Menu` + a shortcode block — not the `wp:navigation` block. `wp:navigation` stores the menu in a `wp_navigation` post and is edited in the Site Editor (a separate entity-save users miss), so edits appear not to "take". Register nav locations, render `wp_nav_menu(['theme_location'=>…,'walker'=>new Theme_Nav_Walker()])` via a shortcode, and drop `[shortcode]` into the header/footer template part with a `wp:shortcode` block. Then it's managed from Appearance → Menus.
- (2026-06-04) WP core's `core/shortcode` block render callback is literally `return wpautop($content);` (wp-includes/blocks/shortcode.php). When a shortcode returns block-level HTML (e.g. a `<nav>` menu), wpautop wraps it in a `<p>` that the browser splits at the block children, injecting a stray empty `<p>` that breaks layout (dropped/misaligned nav). The template-part renderer also runs `do_shortcode()` BEFORE `do_blocks()`, so by the time a `render_block` filter sees the block its innerHTML is already expanded (a token-scoped check fails). Fix: a `render_block` filter that returns `do_shortcode($block['innerHTML'])` (no autop) for `core/shortcode` blocks.
- (2026-06-04) A `position:fixed` mobile nav panel placed inside the header is trapped as a containing block by the header's `will-change: transform` (used for headroom/sticky animation) — `inset:0 0 0 auto` then resolves to the header box, not the viewport, so the panel renders only header-height tall. Size it with viewport units instead: `top:0; right:0; height:100dvh` (with `100vh` fallback).
- (2026-06-04) Header ghost/outline CTA button losing its style: a later, EQUAL-specificity generic rule (e.g. `.wp-block-button.btn--primary .wp-block-button__link`) overrides the header's dedicated CTA rule on source order, repainting it solid (and a transparent border = invisible on a dark header). Scope the header CTA selector with extra specificity (prefix `.site-header`) so it wins. Verify with computed styles, not just visual.
